Flow

Life of a Query

1

Complexity estimation. NLP heuristics score the query 0.0 to 1.0. Simple questions skip debate entirely.

2

Parallel model dispatch. Selected Ollama models receive the prompt concurrently via asyncio.

3

Semantic clustering. Responses grouped by meaning (not string match). Coalitions form before voting.

4

Ensemble voting. Weighted majority + temperature ensemble + confidence-based, then meta-vote.

5

Debate rounds (if needed). Low-confidence results trigger multi-round structured argumentation.

35 Intelligence Improvements

Six categories that make the swarm smarter than any single model

Category 1

Agent Intelligence Patterns

Role Specialization

Planner, Worker, Critic, Judge, Synthesizer. Each gets a tuned system prompt.

ReAct Reasoning

Thought-Action-Observation loops with real tools: web search, calculate, document lookup.

Chain-of-Thought

Force step-by-step reasoning before final answers. Separates thinking from output.

Adaptive Routing

Complexity estimator picks the right pipeline: simple queries skip the heavy machinery.

MoE-Style Routing

Models matched to query type (code, math, creative) based on benchmark strengths.

Goal Alignment

Personality traits map to AgentRoles. Each agent's goals stay consistent across sessions.

Category 2

Voting & Consensus

•

Multi-Algorithm Ensemble. Weighted majority + temperature ensemble + confidence-based, then meta-votes across all three.

•

Confidence Calibration. Tracks predicted vs. actual accuracy. Applies calibration curves so "80% confident" really means 80%.

•

Minority Report. The strongest dissenting view ships alongside consensus. Edge cases don't vanish.

•

Conviction-Weighted Voting. Models holding consistent positions across debate rounds earn higher voting weight.

•

Contextual Algorithm Selection. Query type determines which voting algorithm runs. Factual questions use majority; creative ones use temperature ensemble.

Category 3

Debate & Deliberation

Structured Argumentation

Multi-round debate with pressure prompts. Each round narrows disagreement.

Devil's Advocate

One agent assigned to challenge the emerging consensus. Prevents groupthink.

Evidence-Grounded

Agents must cite sources or reasoning chains. Unsupported claims get flagged.

Stare Decisis

Precedent-based reasoning. Past successful debates inform current deliberation.

Debate Replay

Full transcripts saved to Oracle. Replay any deliberation to audit the reasoning path.

Category 4

Agent Coordination

  • Circuit Breaker: Failing models auto-skipped (CLOSED / OPEN / HALF_OPEN states)
  • Coalition Formation: Similar responses grouped before voting to prevent vote splitting
  • Graceful Degradation: Adapts mode based on available models (full, simplified, single)
  • Adaptive Timeouts: Per-model timeouts based on 3x EMA of historical latency
  • Personality-Driven Comms: Communication style (formal/casual) shapes system prompts
  • Agent Handoff: Capability-based task delegation between agents
Categories 5 & 6

Learning & Observability

Learning & Adaptation

  • Dynamic weight adjustment via EMA win rates
  • User feedback loop (thumbs up/down)
  • A/B testing debate prompts; promotes winners
  • Personality state persists across sessions

Observability

  • Decision explanations auto-generated
  • Performance profiling per model
  • Debate replay with full transcripts
  • Chunk-level RAG attribution

ACP

Agent Communication Protocol

Message Bus

Direct, broadcast, and room channels. Agents subscribe to topics they care about.

Agent Registry

Query agents by role or capability. Dynamic discovery as models come online.

OTP-Style Supervision

Retry with exponential backoff. Crashed agents restart without killing the session.

Role Dispatcher

Maps personality traits to agent roles. Consistent behavior across debate rounds.

Hash Anchoring

Deterministic response references. Every statement gets a content hash for audit trails.

Quick Start

Up and Running in 3 Minutes

1

One-command install

curl -fsSL https://raw.githubusercontent.com/jasperan/ai-congress/main/install.sh | bash
2

CLI mode

python -m ai_congress chat "Compare Python vs Rust for CLI tools"
3

Web interface

cd frontend && npm install && npm run dev # Opens at localhost:5173
Usage

Enhanced Mode (API)

import requests response = requests.post("http://localhost:8000/api/chat/enhanced", json={ "prompt": "Should we use microservices or a monolith?", "mode": "personality", "enable_debate": True, "debate_rounds": 3, "enable_minority_report": True }) result = response.json() print(result["consensus"]) # The winning answer print(result["minority_report"]) # Strongest dissent print(result["confidence"]) # 0.0 - 1.0
